# Brand Voice

## Invariants
- The `T://bendet` namespace: scheme + payload. Eyebrow labels
  (`T://bendet · section`) are identity, not decoration.
- Easter eggs are NEVER labeled, hinted at, or explained. Reward curiosity
  silently. The ones who notice will notice.
- **Never use the phrase "let's connect."** Or its cousins.
- References (HP/LOTR, mythology, anything) are woven, never on-the-nose.
  If a reference needs explaining, it fails.
- A person lives here: anti-template, human warmth against technical precision.
- Honest voice: no promotional framing, no buzzword filler.

## Integrating a new symbolic layer (the Greek mythology rule)
The identity currently carries: developer-protocol (T://bendet) + HP (Marauder's
Map). Adding mythology risks theme soup. Any integration proposal MUST pick one:
1. **Spine** — mythology becomes the organizing metaphor; existing layers are
   recontextualized or superseded (requires ADR flips, incl. possibly 0002)
2. **Subordinate layer** — mythology appears in a bounded register (e.g., naming,
   one typographic gesture) under the existing identity
3. **Rejected** — evaluated honestly and declined, with reasons in an ADR

"All three coexisting equally" is not an option. Mission 1's output contract
requires this reconciliation decision explicitly.
