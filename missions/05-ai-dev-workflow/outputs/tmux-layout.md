# tmux Layout — skipped

Mission 5 · 2026-07-22. Optional deliverable; skipped deliberately.

**Why.** The Phase 2 loop is two long-running processes (the Compose dev
stack and `astro dev`) plus an agent session — which terminal tabs already
handle, and which ADR 0026's one-track-at-a-time rule keeps to a single set.
A prescribed layout would be a personal-preference artifact that nothing else
in the blueprint depends on, and no other document would be different if it
existed. Restraint over decoration is a project invariant; it applies to
workflow machinery too.

Nothing is blocked by this. If a real multi-pane need appears in Phase 2,
it is a preference Tal sets in his own dotfiles, not a decision the workshop
owes an ADR.
